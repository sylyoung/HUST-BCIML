# MSDT

**Axis:** Classical (network-free) &nbsp;·&nbsp; **Paradigm:** MI &nbsp;·&nbsp; **Dataset:** BNCI2014001 (cross-subject LOSO)

**Stage configuration:** `own tangent map · tangent-space features · (no backbone) · per-source MLP + IM adaptation · fit`

## Mechanism
Multi-Source Decentralized Transfer trains one small MLP per source subject on Riemannian tangent-space features (decentralized — no source data is pooled). At transfer time only the source models are shared: the target adapts each source feature extractor (classifiers frozen) by information maximization — minimize per-instance entropy, maximize batch diversity — plus a source-inconsistency term that pulls the sources into agreement, with entropy-based per-source domain weighting; the prediction is the domain-weighted ensemble. It is multi-source and source-free.

## Provenance
- **Paper / source:** Zhang, Wang & Wu, IEEE Trans. Neural Systems and Rehabilitation Engineering 2022 (Multi-Source Decentralized Transfer, MSDT); github.com/chamwen/MSDT — gray-box MSDT-G (target_adapt_msdt_mi), use_weight on
- **Implementation:** Adapted from github.com/chamwen/MSDT (Zhang, Wang & Wu, IEEE TNSRE 2022), the gray-box MSDT-G variant, onto the fit-mode Strategy ABC. Per the LOSO protocol each other subject is one source domain. The upstream mixup is a no-op in the authors' code (its gradients are wiped before the optimizer step) and is omitted here — reproducing the effective optimization, not "fixing" it (disclosed). Made device-agnostic; the tangent map is the only alignment, so the preset aligner is Identity. Requires pyriemann + scikit-learn.

## Note
multi-source classical source-free transfer, lab method. Under LOSO each of the other 8 subjects is one source domain: each trains its own MLP on tangent-space features (decentralized, no source pooling); the target adapts the source extractors (classifiers frozen) by information maximization + a source-inconsistency term with entropy-based domain weighting, and predicts the weighted ensemble. The paper's Table IV on MI2 (2-class 2a, cross-subject LOSO — this benchmark's setup) reports MSDT-G 76.54; the measured 73.33 is -3.21, a faithful port (some upstream hyperparameters are unspecified). The upstream mixup is a disclosed no-op in the authors' code (its gradients are wiped before the optimizer step) and is omitted here, reproducing the effective optimization rather than 'fixing' it. Tangent map is the only alignment, so the preset aligner is Identity.
